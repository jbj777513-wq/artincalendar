## 🏗️ 구축한 자동화 구조

```
사용자가 배포.bat 더블클릭 + 버전 입력 (예: 1.2.3)
        ↓
git add → commit → push → 태그 생성 → 태그 push
        ↓
GitHub Actions 자동 실행 (.github/workflows/build.yml)
        ↓
GitHub 서버(Windows)에서:
  - Python 3.11 설치
  - pip install PyQt5 pyrebase4 requests pyinstaller
  - pyinstaller artincalendar.spec 빌드
  - version.txt에 버전 자동 기록
  - dist 폴더 zip 압축
  - GitHub Releases에 zip 자동 업로드
        ↓
사용자 앱 실행 시 자동 업데이트 팝업 → 업데이트 진행
```

## 📂 추가/수정한 파일

### 1. `.github/workflows/build.yml` (신규)
GitHub Actions 워크플로우. 태그 push 또는 release publish 시 자동 빌드.
- 트리거: `v*` 태그 push, 또는 release published
- `permissions: contents: write` (Release 업로드 권한)
- `pip install PyQt5 pyrebase4 requests pyinstaller`
- zip 이름: `artincalendar-{태그}.zip`

### 2. `.gitignore` (신규)
빌드 결과물 제외: `dist/`, `build/`, `_internal/`, `__pycache__/`, `*.pyc`

### 3. `배포.bat` (신규, artincalendar 폴더)
더블클릭 → 버전 입력 → git add/commit/push/tag 전부 자동.
※ 한글 인코딩 문제로 **영문으로 작성됨** (한글 넣으면 깨짐)

### 4. `updater.py` (버그 수정) ⭐
**중요 버그 수정:** GitHub 릴리즈 `body`(설명)가 비어있으면
`'NoneType' object is not subscriptable` 오류로 업데이트 감지 실패했음.
```python
# 수정 전:  "notes": data.get("body", "")[:300],
# 수정 후:  "notes": (data.get("body") or "")[:300],
```

## ⚠️ 트러블슈팅 기록

| 겪었던 문제 | 원인 | 해결 |
|------------|------|------|
| `requirements.txt not found` | GitHub에 소스 파일 없었음 | 소스 전체 업로드 |
| `artincalendar.spec not found` | 동일 | 동일 |
| `Resource not accessible by integration` | Actions 권한 부족 | `permissions: contents: write` 추가 |
| `src refspec main does not match` | 브랜치명 master였음 | `git branch -M main` |
| 자동업데이트 안 뜸 | 릴리즈 body가 None | `(data.get("body") or "")` 수정 |
| 배포.bat 한글 깨짐 | cmd 인코딩 | bat 내용 영문화 |

**cmd 사용 시 항상 첫 줄:** `cd C:\Users\USER\Desktop\artincalendar`
